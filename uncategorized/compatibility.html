发表于 2016-03-26

透明度

IE6，7下的透明度设置

清除浮动

ie6

H5标签兼容

1.做法可以去document.createElement(“header”)，并在css中display:block；
2.引入html5shiv.js 已经为我们做了相关标签的兼容

IE6 下浮动

块级元素浮动之后，能设置宽度的话就给元素加宽度，如果需要宽度是内容撑开，就需要给里面的块元素加上浮动，否则无效果。
父级浮动，子级没宽度 需要内容撑开就需要也浮动。
第一块元素浮动，第二块元素加margin值等于第一块元素，会有间隙问题。

IE6 子级宽度大于父级

会撑开父级
解决方法: 不要让子元素的宽高超过父级

p标签不能嵌套块级元素

问题会出现 父级的P 会与子级块级元素变为同级级别， 并且 前后都会生成一个P 标签。

margin-top

1.子级的传递
触发BFC，haslayout
父级没margin ，子级有margin 会把margin传递给父级

解决方法：加上zoom：1

2.上下margin叠压
解决方法：尽量使用同一方向的margin

css hack

在css样式写*可以在对应的浏览器识别

*zoom:1

IE 6下 块级最小高度

IE6块级最小高度是19px；
问题描述：加入设置1个1px高度的div ，在IE6下 显示会不止1PX.
解决方法：设置一个overflow:hidden可以解决.

IE6双倍边距问题

当浮动方向与margin方向一致的时候回出现双倍边距
解决方法:
hack
*display:inline

IE6下 li浮动元素4px问题

如果li的子元素浮动， li会出现4px的间隙
解决方法:
hack
*vertical-align:top

IE6下文字溢出的BUG

问题出现:
两个浮动元素中间有内联元素或者注释，并且和父级元素相差不超过3PX.

解决方法:
1.两个浮动元素中间避免出现内联元素或者注释.
2.与父级元素相差超过3px.

IE67 父级ovh包不住 子级加了relative的元素

思路:放他们在同一个环境下.
解决方法:
给父级同样加上relative

IE6下绝对定位元素父级宽高是奇数；绝对定位元素的right，bottom会有1px偏差

解决方法:
避免父级宽高出现奇数

浮动元素和绝对定位元素是同级的话定位元素就会消失

解决方法:
避免 他们同级，可以避免这个bug

IE6 input空隙

解决方法:
给input添加浮动

表单控件的背景图片随着输入内容增多，图片会偏移

解决方法:
background: url()no-repeat fixed；

CSS HACK

简述:针对不同的浏览器写不同的CSS样式的过程就是CSS HACK
书写顺序:先统一版本，再写不同版本的hack

写法:
\9IE10以及IE10以下的
*IE7以及IE7以下的
_IE6以及IE6以下版本的

引入后并且在js中添加代码
